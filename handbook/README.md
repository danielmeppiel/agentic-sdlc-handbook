## How to read this book

**Leaders** (CTO, VP Eng, Chief Architect): Start with Chapter 1, then read Block 1 (Chapters 2-7). Each chapter produces a deliverable — evaluation framework, business case, governance checklist, transition plan.

**Practitioners** (Senior Engineer, Tech Lead, Staff Developer): Start with Chapter 1, then skip to Block 2 (Chapters 8-14). Each chapter teaches a specific discipline with worked examples you can implement Monday morning.

**Both audiences** converge at Chapter 15.
